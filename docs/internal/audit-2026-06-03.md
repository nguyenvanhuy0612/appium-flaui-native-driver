## Resolution (2026-06-03, same day)

**All findings addressed.** Remediation verified on the Windows box: W3C e2e **74/74** (+5 audit tests),
smoke **1/1**, Mac **116** unit + **5** C# scheduler tests.

- **Security (F22/F23/F24):** gates made non-optional & loud; `appium:prerun` now gated by `power_shell`;
  file-transfer trust boundary documented (no sandbox — by design). **Per ADR-015 the posture is permissive
  by default** (isolated VMs; never sacrifice a feature): runs use `appium --relaxed-security`, under which
  every gate passes — the gates only bite if an operator deliberately locks down via scoped `allow-insecure`.
  Re-verified: e2e 74/74 + smoke 1/1 under `--relaxed-security`.
- **Stability (F1/F3/F4/F5/F7/F10/F11):** scheduler serialized + atomic poison + fatal budget; PowerShell
  child bounded (timeout + kill-tree + concurrent IO); TS layer-5 recycle+reattach+retry-once on transport
  failure; COM released on registry eviction; app Kill fallback + `ms:forcequit`; `flaui:*` caps threaded.
- **Protocol (F16/F17/F18/F20):** mapped W3C errors for malformed JSON/op/non-JSON; hex HWND →
  invalid argument; xpath empty → no such element; setWindowRect Win32 fallback.
- **Docs honesty (F14/F15/F19/F26):** page source documented as LIVE (cached pass = TODO); uia2 marked
  experimental; rawView accepted-but-ignored; ✅/🟡 reconciled.

**Deferred (proofs only, not code):** F1 live frozen-app hang-injection E2E and the 30-min session-stress
(stress deferred by the user) — the recycle/poison code paths are implemented and unit-covered.
**Accepted as-is:** F8 (FIFO registry — fine at real traffic), F21 (single window handle, v1).
